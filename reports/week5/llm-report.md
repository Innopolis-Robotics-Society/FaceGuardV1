# LLM Usage Report

This Week 5 repository update used LLM tools, primarily Codex, as an assistant
for repository analysis, drafting, consistency checks, and implementation
support.

## Where LLM Support Was Used

- Read and summarized the Assignment 5 requirements and mapped them to the
  existing repository structure.
- Inspected the repository to identify Sprint 3 / MVP v2 issues, user stories,
  bugs, PR evidence, release evidence, CI evidence, and remaining report gaps.
- Drafted and refined Week 5 report sections, including MVP v2 status, customer
  feedback response, screenshot evidence, UAT results, Sprint Review summary,
  retrospective, reflection, and contribution traceability.
- Helped describe the project structure and maintain links between the root
  README, roadmap, Definition of Done, testing documentation, UAT
  documentation, architecture documentation, and Week 5 report.
- Assisted with architecture documentation wording, including static, dynamic,
  and deployment views, ADR traceability, and PlantUML/UML diagram context.
- Helped update UAT scenarios for service monitoring, recognition score
  display, unavailable-camera behavior, and MVP v2 customer validation.
- Helped identify stale `Pending` placeholders and distinguish completed public
  evidence from private Moodle-only evidence.
- Helped prepare changelog wording for MVP v2 changes, including service
  status monitoring, recognition confidence handling, model management,
  documentation, testing, and screenshot evidence.

## Human Review and Responsibility

All LLM-generated or LLM-assisted materials were reviewed, edited, and accepted
by the team before inclusion. The team remained responsible for factual
accuracy, repository changes, customer-meeting interpretation, screenshots,
release links, private evidence handling, and final submission decisions.
