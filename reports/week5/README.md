# Week 5 Report

## Main result

Week 5 delivers the team-facing process guide requested in issue `#60`:

- [Development and configuration process](../../docs/development-process.md)

## What the guide covers

The new document records the workflow the team uses to:

- refine backlog items and plan Sprint work;
- track issue status on the GitHub Project board;
- create branches from `main` and move work through review;
- require an independent reviewer and passing CI before merge;
- update releases and `CHANGELOG.md` from reviewed work on `main`;
- configure the frontend, backend, database, and recognition agent;
- separate public development defaults from private secrets.
