# Assignment 4 Submission Checklist

Use this checklist before final public GitHub and private Moodle submission.
Leave unchecked any item without evidence.

## GitHub Before Submission

- [ ] All selected PRs are merged.
- [ ] Selected Issues close automatically through linked PR merge.
- [ ] Latest `main` CI is green.
- [ ] Latest link check is green.
- [ ] Milestone reflects completion.
- [ ] Assignment 4 release is published.
- [ ] Deployment or runnable artifact is verified.
- [ ] Week 4 report is updated with final links.
- [ ] Evidence images are committed under `reports/week4/images/`.
- [ ] Public demo is available and sanitized.
- [ ] No credentials, `.env` files, biometric images, customer data, or private
  evidence are committed.

## Customer Session

- [ ] Recording consent is confirmed.
- [ ] Sprint Review is completed.
- [ ] UAT-001 is executed.
- [ ] UAT-002 is executed.
- [ ] UAT-003 is executed.
- [ ] UAT-004 is executed if Dashboard refresh is in the final demo scope.
- [ ] Notes or transcript are prepared according to publication permission.
- [ ] Timecodes are captured.
- [ ] Feedback response table is updated.

## Moodle PDF

- [ ] Team table is included.
- [ ] Contributions are summarized.
- [ ] Non-participation statement is included if applicable.
- [ ] Commit permalink to Week 4 README is included.
- [ ] Commit permalink to repository tree is included.
- [ ] Customer recording is attached or linked privately.
- [ ] UAT timecodes are included.
- [ ] Credentials/access details are included privately if required.
- [ ] Rehearsal video is included.
- [ ] Instructor-only evidence is included.

## Slides

- [ ] Final deck is prepared.
- [ ] One team submission is selected.
- [ ] Rehearsal is complete.
- [ ] Presentation evidence uses only public or sanitized material.
